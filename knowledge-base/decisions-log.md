# Decisions Log — the learning loop

> Every time Adam reacts to a role ("yes, that fits" / "no, not that"), record it here.
> Before each new `/find-roles` sweep, read this file — it sharpens targeting over time.
> `/log-decision` appends entries; `/refine-search` rolls patterns up into `fit-criteria.md`.

## Format
```
### YYYY-MM-DD — [Firm / Role or pattern]
- **Verdict:** FIT / NO-FIT / MAYBE
- **What Adam said:** (his actual words/reaction)
- **Why:** (the underlying signal)
- **Criteria implication:** (what to weight up/down, any new gate)
```

## Rolled-up patterns (maintained by /refine-search)
- **Bullseye archetype:** lean, AI-forward fund; generalist seat blending **fund operations + diligence + LP/IR**; direct principal proximity; reachable hub (Dallas/Austin) or remote. (SPLY = the exemplar.)
- **Carry is not a filter.** Don't gate or score on it; raise it in conversation.
- **Firm caliber can beat title.** Flex down on title for a top-tier firm with a real track up.
- **Domain-pedigree requirements are real screens.** A great-shaped role in a sector he lacks pedigree in (e.g. healthcare-VC-trained) is a likely pass — discount it.
