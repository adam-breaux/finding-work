You are running Adam's job-search sweep. Goal: find real open roles that fit, score them, and record them. Do NOT auto-apply.

## Steps
1. **Load context.** Read `knowledge-base/profile.md`, `knowledge-base/fit-criteria.md`, and `knowledge-base/decisions-log.md` (especially rolled-up patterns) so the search reflects the latest learning. Note any focus the user passed: "$ARGUMENTS".
2. **Search broadly.** Use web search across, at minimum:
   - VC talent boards & fund career pages (John Gannon's VC job board, Getro/Consider talent networks, individual fund "Careers" pages)
   - LinkedIn Jobs (Senior Associate / Principal / Investor / Chief of Staff at VC, CVC, family office, PE)
   - Family-office and CVC openings; PE associate/principal roles
   - Hybrid roles anchored in Dallas / Austin / Denver (Adam's acceptable hubs) and remote-friendly funds
   Prefer roles posted/active recently. Pull the firm, title, location/model, comp signals, and a link.
3. **Score each** 0–100 with `fit-criteria.md`. Apply dealbreaker gates. Flag ⭐ if a warm-intro path exists (cross-check `knowledge-base/network.md`).
4. **Record.** Append the qualifying roles (score ≥ 40, or anything notable) to the table in `knowledge-base/opportunities.md`, newest first, with a one-line note. For the strongest (≥ 80) or any the user wants, write a detailed block too.
5. **Report.** Give the user a ranked shortlist: firm — role — location/model — score — the one reason it fits and the one risk. Offer next actions (`/score-role`, `/tailor-resume`, `/draft-outreach`).

## Rules
- Real, verifiable roles only — include links. If you can't confirm a posting is live, say so and mark it `UNVERIFIED`.
- Be honest about thin results; a short sweep is fine. Quality over quantity.
- Don't fabricate comp; mark unknowns as such.
- Respect the geography reality: Tulsa-anchored, hybrid-to-a-hub or remote; never surface relocation-required roles as fits.
- Commit the updated `opportunities.md` when done.
